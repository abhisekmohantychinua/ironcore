Iron Core - Project Standards & Coding Guidelines

Principles

  • Keep it simple, consistent, and discoverable. Prefer clarity over cleverness.
  • Prioritize semantic HTML and progressive enhancement. JS enhances interactions, not baseline functionality.
  • Accessibility, readability, and performance are first-class concerns. Follow WCAG principles.
  • Prefer widely-used conventions to reduce cognitive load.

HTML Standards

  • Use semantic tags: <header>, <main>, <nav>, <section>, <article>, <footer>, headings hierarchically.
  • One <h1> per page; use <h2>, <h3> for subheadings.
  • Every image must have alt attribute; decorative images alt="".
  • Use <a> for navigation and <button> for actions.
  • Forms must have <label> or aria-label and validation.
  • Use ARIA only when necessary, e.g., modals: aria-hidden, aria-modal="true".

CSS Standards

  • Keep CSS atomic and component-scoped; utility classes sparingly.
  • Use variables.css for tokens; refer consistently: var(--color-primary).
  • BEM-like naming: .component, .component__element, .component--modifier.
  • Mobile-first, min-width media queries; use clamp() for fluid typography.
  • Use Grid for layout, Flexbox for components; avoid layout margins.
  • Respect prefers-reduced-motion and visible focus styles.

JavaScript Standards

  • ES6+: const/let, arrow functions, modules.
  • Follow consistent style (Airbnb recommended), const by default.
  • Use async/await; try/catch for async code.
  • Event delegation for lists/grids; batch DOM updates.
  • Debounce/throttle high-frequency events; sanitize innerHTML.
  • Wrap code in modules or namespaces; avoid globals.
  • Feature detection & progressive enhancement.
  • Trap focus in modals; provide keyboard handlers for interactive elements.

Accessibility (A11y)

  • WCAG 2.1 AA compliance.
  • Keyboard support for all interactive elements.
  • Color contrast and skip-links.
  • Descriptive labels and inline error messaging in forms.

Performance & Assets

  • Optimized images (AVIF/WebP), lazy-load images/videos.
  • Minify CSS/JS, caching, hashed filenames.
  • Limit fonts; prefer system fonts.

Security & Privacy

  • No secrets in client-side code.
  • Sanitize and validate inputs server-side.
  • Apply basic CSP headers; keep libraries updated.

JS Module Pattern (ESM)

export function openModal(modalEl) {
  modalEl.setAttribute('aria-hidden', 'false');
}
export function closeModal(modalEl) {
  modalEl.setAttribute('aria-hidden', 'true');
}
